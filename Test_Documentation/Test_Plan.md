# Project: Foodie Sizzle

Platform: iOS

Tester: QA Engineer (Anastasiia Sysoieva)

## Objective
    
The purpose of testing is to verify that gameplay mechanics, rewards, life system, and user interactions work correctly and provide a stable player experience.
    
## Test Scope

In Scope:

- Level gameplay
- Timer behavior
- Life (energy) system
- Customer order mechanics
- Daily rewards
- Competitions/events
- In-game purchases
- UI interactions and animations

Out of Scope:

- Backend implementation
- Payment provider internal processing
- Developer admin tools
## Test Types
- Functional testing
- UI testing
- Usability testing
- Smoke testing
- Regression testing
- Exploratory testing
- Compatibility testing (iOS devices)
## Test Environment
- Platform: iOS mobile devices
- Network: Wi-Fi / Mobile Internet / Offline scenarios
- Game version: current production build
## Entry Criteria
- Game installed successfully
- Application launches without crash
- Test account available
- Stable internet connection available (for online features)
## Exit Criteria
- No Critical or Blocker defects remain open
- Major gameplay flows verified
- ≥95% test cases passed
- Regression testing completed
## Reports
- Test cases
- Checklists
- Bug reports in Jira
- Test execution results
## Risks
- Internet instability affecting gameplay
- Data synchronization issues
- Incorrect reward calculations
- Life system inconsistencies
## Test Schedule
Testing is performed continuously during gameplay sessions and after game updates.
